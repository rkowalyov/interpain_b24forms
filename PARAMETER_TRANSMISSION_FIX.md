# 🔧 РЕШЕНИЕ: Передача EVPASSPORT_ID в Bitrix

## 🎯 Обнаруженная проблема

**Симптомы:**
- Параметр EVPASSPORT_ID передаётся ✅
- Но в Bitrix приходит значение **0** ❌

**Корневая причина:**
Параметр `EVPASSPORT_ID` в форме Bitrix настроен как тип поля **"Целое число"** (Integer)

## ✅ Решение

### Если поле настроено как "Целое число"

**Вариант 1 (Рекомендуется): Изменить в Bitrix CRM**
1. Откройте администратор Bitrix: https://вашдомен.bitrix24site.ru/admin/
2. Перейдите в **CRM** → **Каналы** → **Формы**
3. Найдите вашу форму и нажмите **Редактировать**
4. Найдите поле **EVPASSPORT_ID**
5. **Измените тип** с "Целое число" на **"Текст"** или **"Строка"**
6. Сохраните изменения

**Результат:** Параметр будет передан как текст и буду сохранен корректно ✅

### Вариант 2: Если тип не должен меняться

Если поле ДОЛЖНО оставаться числовым, убедитесь что:
1. Значение передается как **правильное число без спецсимволов**
2. В URL используется формат: `?EVPASSPORT_ID=845` (только цифры)
3. Параметр НЕ содержит пробелы или другие символы

**Текущий код:**
- ✅ Парсит значение как число: `parseInt(idStr, 10)`
- ✅ Проверяет валидность: `isNaN(id)`
- ✅ Передает как число в FormData

## 🧪 Тестирование

### Проверка формата параметра

```bash
# Правильно (только цифры):
?EVPASSPORT_ID=845
?EVPASSPORT_ID=229

# Неправильно (будет ошибка):
?EVPASSPORT_ID=0845        (ведущий ноль)
?EVPASSPORT_ID=845a        (буква)
?EVPASSPORT_ID=845%20229   (пробел/символ)
```

### Проверка в консоли

Откройте F12 → Console и посмотрите строку:
```
ℹ️ EVPASSPORT_ID received: 845 (parsed as number: 845)
```

Если вместо этого видите:
```
❌ Invalid EVPASSPORT_ID: null (parsed as: null)
```

Значит параметр не передан вообще - проверьте URL.

## 📝 Почему это происходит

Bitrix имеет **типизированные поля**:

| Тип поля | Что можно передать | Что происходит | Проблема |
|----------|-------------------|----------------|----------|
| Целое число | `123`, `845` | ✅ Сохраняется как число | Если приходит строка - может быть ошибка преобразования → 0 |
| Текст / Строка | `"текст"`, `"845"` | ✅ Сохраняется как текст | Никаких проблем |
| Число (Float) | `123.45` | ✅ Сохраняется как число | Целые числа преобразуются правильно |

## 🎯 Рекомендуемые шаги

1. **Сейчас:** Откройте форму в Bitrix администраторе и посмотрите тип поля EVPASSPORT_ID
2. **Действие:** Измените тип на "Текст" (если только это не критично)
3. **Тест:** Перейдите по ссылке `?EVPASSPORT_ID=845` и заполните форму
4. **Проверка:** Посмотрите логи в консоли (F12)
5. **Результат:** Значение должно появиться в Bitrix CRM (не как 0)

## ❓ Дополнительно

Если тип поля менять нельзя, свяжитесь с поддержкой Bitrix - может быть специальный формат передачи числовых параметров через формы.
