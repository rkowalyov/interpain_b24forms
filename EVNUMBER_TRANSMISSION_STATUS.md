## ❌ ПРОБЛЕМА

Параметр **EVNUMBER** может приходить в Bitrix как **0** из-за:

### Возможная причина 1: Поле настроено как INTEGER
Если в Bitrix форме поле EVNUMBER настроено как **"Целое число"**, Bitrix может неправильно преобразовать строковое значение.

**Решение:** Изменить тип поля на "Текст" в админке Bitrix

### Возможная причина 2: Неправильное имя поля
Поле в Bitrix может называться не `EVNUMBER`, а иначе.

**Возможные названия:**
- `EVPASSPORT_ID`
- `PASSPORT_ID` 
- `UF_EVNUMBER` (с префиксом UF для кастомных полей)
- Другое название

**Решение:** Проверить в админке Bitrix → Форма → Поля → Свойства → "Идентификатор поля"

### Возможная причина 3: Параметр не передается в правильном формате
Для числовых полей нужно передавать БЕЗ URL-кодирования.

**Текущая реализация:**
```javascript
// ✅ Правильно (без кодирования)
modifiedUrl = modifiedUrl + '?EVNUMBER=' + 845  // просто число

// ✅ FormData (может быть строка)
body.append('EVNUMBER', String(845))  // как строка - нормально
```
