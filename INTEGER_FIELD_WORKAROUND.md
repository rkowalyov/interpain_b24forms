# 🔢 Решение: EVPASSPORT_ID как числовое поле

## Проблема

Если поле `EVPASSPORT_ID` в Bitrix настроено как **"Целое число"** и вы **не можете изменить тип**, нужно использовать специальный подход.

## ✅ Решение

Есть несколько способов передачи параметра к числовому полю:

### Способ 1: Через URL параметры (РАБОТАЕТ)
```
https://interpain-b24forms.vercel.app/event-register/?EVPASSPORT_ID=845
```

**Как это работает:**
- Параметр передается БЕЗ URL encoding (просто число: `845`, не `%845`)
- Bitrix получает значение как число в URL
- Значение попадает в числовое поле как `845` (не как строка)

### Способ 2: Проверить название поля в Bitrix
Возможно, поле в Bitrix названо не `EVPASSPORT_ID`, а иначе:
- `EVNUMBER` 
- `PASSPORT_NUMBER`
- `PASSPORT_ID`
- `UF_EVPASSPORT_ID` (с префиксом UF)
- Что-то другое

**Как узнать:**
1. В Bitrix администраторе откройте форму
2. Нажмите на свойства поля
3. Посмотрите **"Идентификатор поля"** (Field ID)
4. Это и есть название, которое нужно использовать

### Способ 3: Использовать прямую ссылку на форму Bitrix
Если форма доступна по прямой ссылке в Bitrix:

```
https://вашдомен.bitrix24site.ru/ваша_форма/?EVPASSPORT_ID=845
```

Так параметр передается напрямую Bitrix без промежуточных преобразований.

## 🔍 Отладка

### Проверка 1: Убедитесь что число передается
В консоли браузера (F12 → Console) должно быть:
```
ℹ️ EVPASSPORT_ID received: 845 (parsed as number: 845)
```

### Проверка 2: Убедитесь что число в URL не encoded
```
✅ Правильно: 
https://.../?EVPASSPORT_ID=845

❌ Неправильно:
https://.../?EVPASSPORT_ID=%20845 (пробел)
https://.../?EVPASSPORT_ID=0845 (ведущий ноль)
https://.../?EVPASSPORT_ID=%68 (hex encoding)
```

### Проверка 3: Смотрите логи отправки
В консоли при отправке формы ищите:
```
🔗 Added EVPASSPORT_ID to URL as number: 845 (direct value, no encoding)
✅ Added params to FormData (EVPASSPORT_ID as string: 845, type: string)
```

Если этого нет - параметр не передается.

## 📋 Дополнительные методы

### Если всё равно 0 - попробуйте альтернативные названия

Код автоматически пытается использовать несколько вариантов названий полей. Проверьте логи для:
```
🔍 Checking for potential EVPASSPORT fields...
  Found passport field: [ИМЯ_ПОЛЯ], filling with 845
```

### Используемые поля при отправке:
- **В URL:** `?EVPASSPORT_ID=845` (как число, без encoding)
- **В FormData:** `EVPASSPORT_ID: "845"` (как строка - это нормально, Bitrix сам преобразует)
- **Альтернативные поля:** Поиск полей с названиями содержащими "passport"

## ⚙️ Что происходит в коде

1. **При загрузке страницы:**
   - Извлекается параметр `EVPASSPORT_ID` из URL
   - Парсится как число: `parseInt(idStr, 10)`
   - Сохраняется в переменную `id`

2. **При загрузке Bitrix формы:**
   - Параметр добавляется в URL без encoding (чистое число)
   - Параметр добавляется в FormData (как строка - нормально)
   - Параметр добавляется через API методы формы если доступны
   - Ищутся скрытые поля и заполняются

3. **При отправке формы:**
   - Перехватывается POST запрос
   - Проверяется URL и FormData
   - Добавляются параметры перед отправкой
   - Отправляется на Bitrix

## 🎯 Главное правило

**Для числовых полей в Bitrix:**
- URL: передавайте как число без кавычек и символов: `?EVPASSPORT_ID=845`
- FormData: может быть как строка или число - Bitrix сам разберется
- Важно: НЕ использовать encoding для URL параметров числовых полей

## ❌ Если всё ещё не работает

1. **Уточните точное название поля** в админке Bitrix
2. **Скопируйте логи консоли** (F12) и отправьте их
3. **Проверьте логи на Bitrix стороне** - может быть там другой формат требуется
4. **Свяжитесь с поддержкой Bitrix** - они помогут определить правильный способ передачи
